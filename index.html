Web Content Accessibility Guidelines - standards for making web content accessible.

What NVDA announces:

  • For unordered lists: "List with X items" at start, "bullet" before each item, "out of list" at end
  • For ordered lists: "List with X items" at start, "1." (the actual number) before each item, "out of list" at end
  • For definition lists: Typically announces "term" and "definition" pairs

Forms

What NVDA announces:

  • Text inputs: "Edit, [label text]"
  • Password: "Edit, password, [label text]"
  • Checkbox: "Checkbox not checked, [label text]" or "Checkbox checked, [label text]"
  • Radio: "Radio button not checked, [label text]" or "Radio button checked, [label text]"
  • Select: "Combo box, [label text]"
  • Button: "Button, [button text]"
  • Required fields: Typically announces "required" after the control type

Landmarks & Regions

HTML5 semantic elements and ARIA landmarks help screen reader users navigate the page structure.

Article Element

This is content inside an article element, which represents a self-contained composition.

Aside Element

This is content inside an aside element, which represents secondary content indirectly related to the main content.

Section with aria-label

This section has a custom label that screen readers will announce.

DIV with region role

This div uses the ARIA role "region" to mark it as a significant section.

What NVDA announces:

  • Article: "Article" landmark
  • Aside: "Complementary" landmark
  • Section with aria-label: "[label text], region"
  • Other landmarks: "Banner" (header), "Navigation", "Main", "Contentinfo" (footer)

Screen reader users can navigate between landmarks using the 'D' key in NVDA.
